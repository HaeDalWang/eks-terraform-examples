Terraform을 사용한 EKS 프로비저닝

사용자 지정 VPC 네트워킹 및 관리형 노드 그룹을 사용하여 여러 가용 영역(AZ)에 걸쳐 Amazon Elastic Kubernetes Service(EKS)를 구성하는 엔터프라이즈급 구현 가이드입니다.

01 핵심 설계 원칙

고가용성

제어 평면의 자동 장애 조치를 위해 3개의 서브넷에 걸친 멀티 AZ 배포를 지원합니다.

보안 우선

프라이빗 엔드포인트 액세스만 허용하며, 서비스 계정용 IAM 역할(IRSA)이 통합되어 있습니다.

구현 상세

이 블루프린트는 공식 terraform-aws-modules/eks/aws 모듈 20.0+ 버전을 활용합니다. 클러스터 구성에 대한 세밀한 제어력을 유지하면서 보안 그룹 규칙 및 OIDC 프로바이더 설정의 복잡성을 추상화합니다.

warning 요구 사항
  • • Terraform >= 1.3.0
  • • AWS Provider >= 5.0
  • • kubectl 로컬 설치 필요
main.tf 

module "eks" {
  source  = "terraform-aws-modules/eks/aws"
  version = "~> 20.0"

  cluster_name    = "blueprint-cluster"
  cluster_version = "1.29"

  vpc_id     = module.vpc.vpc_id
  subnet_ids = module.vpc.private_subnets

  cluster_endpoint_public_access = false

  eks_managed_node_groups = {
    core = {
      min_size     = 2
      max_size     = 5
      desired_size = 3

      instance_types = ["t3.large"]
      capacity_type  = "ON_DEMAND"
      
      labels = {
        Environment = "production"
        ManagedBy   = "Terraform"
      }
    }
  }

  # AWS Auth Config
  manage_aws_auth_configmap = true
}
